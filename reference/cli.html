Command-line Interface#

bumpversion#

Version bump your Python project.

bumpversion [OPTIONS] COMMAND [ARGS]...

Options

--version#

Show the version and exit.

bump#

Change the version.

ARGS may contain any of the following:

VERSION_PART is the part of the version to increase, e.g. minor . Valid values include those given in the –serialize / –parse option.

FILES are additional file(s) to modify. If you want to rewrite only files specified on the command line, use with the –no-configured-files option.

bumpversion bump [OPTIONS] [ARGS]...

Options

--config-file <FILE>#

Config file to read most of the variables from.

-v, --verbose#

Print verbose logging to stderr. Can specify several times for more verbosity.

--allow-dirty, --no-allow-dirty#

Don’t abort if working directory is dirty, or explicitly abort if dirty.

--current-version <VERSION>#

Version that needs to be updated

--new-version <VERSION>#

New version that should be in the files

--parse <REGEX>#

Regex parsing the version string

--serialize <FORMAT>#

How to format what is parsed back to a version

--search <SEARCH>#

Template for complete string to search

--replace <REPLACE>#

Template for complete string to replace

--no-configured-files#

Only replace the version in files specified on the command line, ignoring the files from the configuration file.

--ignore-missing-version#

Ignore any Version Not Found errors when searching and replacing in files.

-n, --dry-run#

Don’t write any files, just pretend.

--commit, --no-commit#

Commit to version control

--tag, --no-tag#

Create a tag in version control

--sign-tags, --no-sign-tags#

Sign tags if created

--tag-name <TAG_NAME>#

Tag name (only works with –tag)

--tag-message <TAG_MESSAGE>#

Tag message

-m, --message <COMMIT_MSG>#

Commit message

--commit-args <COMMIT_ARGS>#

Extra arguments to commit command

--list#

List machine readable information

replace#

Replace the version in files.

FILES are additional file(s) to modify. If you want to rewrite only files specified on the command line, use with the –no-configured-files option.

bumpversion replace [OPTIONS] [FILES]...

show#

Show current configuration information.

bumpversion show [OPTIONS] [ARGS]...

Options

--config-file <FILE>#

Config file to read most of the variables from.

-f, --format <format_>#

Config file to read most of the variables from.

Options:

default | yaml | json

-i, --increment <increment>#

Increment the version part and add new_version to the configuration.

Arguments

ARGS#

Optional argument(s)

Environment variables

BUMPVERSION_CONFIG_FILE

Provide a default for --config-file

BUMPVERSION_FORMAT

Provide a default for -f

BUMPVERSION_INCREMENT

Provide a default for -i
